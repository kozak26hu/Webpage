\chapter{Executive Summary}

Redefining the ``paper hands'' meme, \$PHANDS offers high-yield staking (20\% APY, 30-day lockup, 180-day cycle), 5\% referral bonuses on user activity (e.g., swaps, minted as new tokens, max. 1,000 PHAND/referrer/tx), and 20\% deposit bonuses for the first 5 liquidity providers (min. 1,000 PHAND swap, max. 200 PHAND bonus) in WETH/PHANDS pools (\$\,319 current liquidity, targeting \$\,5,000--\$\,50,000 short-term, \$\,100,000 long-term). With an initial 1,000,000 \$PHANDS supply (mintable for airdrops/referrals, planned to be fixed in Q2 2026 via ownership renounce), and anti-whale limits (2\% max wallet, 1\% max tx), the project drives adoption via airdrops (100 PHAND for the first 10 Telegram joiners). This whitepaper details the token’s mechanics, economics, and roadmap to achieve 10,000+ holders by Q3 2026. \chapter{Technology}
\$PHANDS is a Solidity v0.8.30 ERC-20 token using OpenZeppelin libraries \par (ERC20, ERC20Pausable, Ownable),  deployed at  \par \small\url{https://etherscan.io/address/0x11157Da1fC6dCfd58b50ed79082183b2c6176245}\normalsize. A separate staking contract enables high-yield rewards and liquidity provider (LP) bonuses, verified on Etherscan.

\begin{itemize}
    \item \textbf{Architecture:} ERC-20 with custom \texttt{\_update} hooks for limits and referrals; staking contract with SafeERC20, ReentrancyGuard, Pausable.
    \item \textbf{Consensus:} Ethereum Proof-of-Stake (PoS) for secure, energy-efficient operations.
    \item \textbf{Key Functions (Token Contract):}
    \begin{itemize}
        \item Transfers with 1\% max transaction (10,000 PHAND) and 2\% max wallet (20,000 PHAND, except owner).
        \item Referral system: \texttt{setReferrer()} and \texttt{claimReferralBonus()} (5\% bonus on user activity, e.g., swaps, minted as new tokens, max. 1,000 PHAND/referrer/tx).
        \item Airdrop: Batch distribution to max. 20 addresses per call (e.g., 100 PHAND for first 10 Telegram joiners).
        \item Pause/unpause for emergency control (renounceable).
    \end{itemize}
    \item \textbf{Key Functions (Staking Contract):}
    \begin{itemize}
        \item 30-day lockup (\texttt{LOCK\_PERIOD = 30 days}).
        \item 20\% annual APY (\texttt{REWARD\_RATE = 55}, $\sim$0.055\% daily, adjusted by stake ratio, 180-day cycle).
        \item LP staking with 20\% deposit bonus for WETH/PHANDS pools.
        \item Auto-claim option for rewards.
    \end{itemize}
    \item \textbf{Security:} Reentrancy protection, immutable limits, owner-controlled minting. Audit planned (Certik, Q1 2026).
    \item \textbf{Scalability:} Ethereum Mainnet (15 TPS); planned Layer 2 migration (e.g., Polygon, 2,000+ TPS) for lower fees and faster transactions.
    \item \textbf{Liquidity Pools:}
    \begin{itemize}
        \item Uniswap V2/V3: WETH/PHANDS pool ($\sim$\$\,110 liquidity, under development, targeting \$\,5,000--\$\,50,000 short-term, \$\,100,000 long-term).
        \item Balancer V3: WETH/PHANDS pool ($\sim$\$\,209 liquidity, under development, targeting \$\,5,000--\$\,50,000 short-term, \$\,100,000 long-term).

\section{Sample Code (Token Contract)}
\begin{lstlisting}
// SPDX-License-License: MIT
pragma solidity ^0.8.20;
import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
import "@openzeppelin/contracts/token/ERC20/extensions/ERC20Pausable.sol";
import "@openzeppelin/contracts/access/Ownable.sol";

contract PaperHands is ERC20, ERC20Pausable, Ownable {
    uint256 public immutable maxTxAmt = 10000 * 10 ** 18; // 1% limit
    uint256 private immutable mwp = 200; // 2% max wallet
    uint256 private immutable rbp = 500; // Referral bonus 5%
    mapping(address => address) public referrers;

    constructor() ERC20("PaperHands", "PHANDS") Ownable(msg.sender) {
        _mint(msg.sender, 1000000 * 10 ** 18); // Initial 1M supply
    }

    function mint(address to, uint256 amt) public onlyOwner {
        _mint(to, amt);
    }

    function setReferrer(address ref) external {
        require(referrers[msg.sender] == address(0), "Ref already set");
        require(ref != msg.sender, "No self-ref");
        referrers[msg.sender] = ref;
    }

    function claimReferralBonus(address buyer, uint256 buyAmt) external onlyOwner {
        address ref = referrers[buyer];
        if (ref != address(0)) {
            uint256 bonus = (buyAmt * rbp) / 10000;
            _mint(ref, bonus);
        }
    }

    function airdrop(address[] calldata recips, uint256[] calldata amts) external onlyOwner {
        require(recips.length == amts.length && recips.length <= 20, "Array mismatch or too long");
        for (uint i = 0; i < recips.length; i++) {
            _mint(recips[i], amts[i]);
        }
    }

    function pause() public onlyOwner { _pause(); }
    function unpause() public onlyOwner { _unpause(); }

    function _update(address from, address to, uint256 amt) internal override(ERC20, ERC20Pausable) {
        if (from != address(0)) {
            require(amt <= maxTxAmt, "Exceeds tx limit");
            if (to != owner()) {
                uint256 totSup = totalSupply();
                uint256 maxWal = (totSup * mwp) / 10000;
                require(balanceOf(to) + amt <= maxWal, "Exceeds wallet limit");
            }
        }
        super._update(from, to, amt);
    }
}
\end{lstlisting}

\chapter{Risks and Mitigation}
\begin{itemize}
    \item \textbf{Market Risk:} Memecoin volatility -- Mitigated by 20\% APY and 5\% referral incentives.
    \item \textbf{Technical Risk:} Contract bugs -- OpenZeppelin libraries; Certik audit planned (Q1 2026).
    \item \textbf{Regulatory Risk:} EU MiCA compliance -- Transparent design, legal consultation ongoing.
    \item \textbf{Adoption Risk:} Slow growth -- Airdrops (100 PHAND for first 10 Telegram joiners) and 5\% referrals for viral spread.
    \item \textbf{Inflation Risk:} Owner-controlled minting for referrals/airdrops (max. $\sim$3--4\% supply increase, e.g., 25,000 PHANDS for 25 referrers at 1,000 PHAND each) -- Mitigated by owner-controlled limits, planned ownership renounce in Q2 2026, and potential long-term governance (2027+).
\end{itemize}
